## 二、内置工具

SmartClaw 在启动时，除了加载用户自定义的 Skills 外，必须内置以下 8 个核心基础工具 (Core Tools)。根据“优先使用 LangChain 原生工具”的原则，技术选型更新如下：

### 1. 命令行操作工具 (Command Line Interface)

- **功能描述**: 允许 Agent 在受限的安全环境下执行 Shell 命令。

- **沙箱方案**:
  - 使用 Docker 容器隔离，会话级持久容器（每个会话创建一个容器，会话期间复用）
  - 基础镜像：`alpine:3.19`（极小，仅 5MB）

- **目录挂载**:
  - 用户指定的 `root_dir` 挂载到容器的 `/workspace`
  - 其他路径不可见，实现文件系统隔离

- **用户权限**:
  - 容器内使用非 root 固定用户（UID 1000）

- **网络隔离**:
  - 允许出站网络（支持 git、curl、pip 等操作）

- **容器启动方式**:
  - 使用 Docker SDK（`python` 的 `docker` 包）

- **容器生命周期管理**:
  - 会话级持久容器：每个会话创建一个容器，会话期间复用
  - 容器创建时机：会话开始时（首次调用 terminal 工具时）
  - 容器关闭时机：仅在会话结束时（用户关闭对话窗口、切换会话、删除会话等）
  - 不设置超时机制

- **容器关闭信号**:
  - 先发送 `SIGTERM` 信号优雅关闭（等待 10 秒）
  - 超时后发送 `SIGKILL` 信号强制终止

- **命令确认机制**:
  - 在容器执行**之前**检查命令
  - 使用完整命令解析（处理 `sudo rm` 等组合命令）
  - 所有需要确认的命令都需要用户确认，禁止直接跳过

- **命令分类**:

  | 类型 | 命令 | 处理方式 |
  |------|------|----------|
  | **直接禁止** | dd, mkfs, sudo, reboot, shutdown, poweroff, halt, ssh, scp, sftp, rsync, \| sh, \| bash, \| python, fork bomb (:(){ :\|:& };:), insmod, rmmod, modprobe, chroot, mount, umount | 返回拒绝提示 |
  | **每次确认** | rm, mv (覆盖), cp (覆盖), chmod, chown, git push, git reset --hard, git rebase, pip install, npm install, yarn install, brew install, docker run, docker exec, crontab, ln -s | 弹窗请求用户确认 |
  | **正常执行** | ls, cat, grep, echo, python, git add, git commit, git clone, git pull, git status, mkdir, touch, head, tail, wc, find, sed, awk 等 | 直接执行 |
  | **网络请求** | curl, wget, git push 等 | 检测到数据上传时需要确认 |

- **资源限制**:
  - 内存：256MB
  - CPU：25%（cpu_quota=25000, cpu_period=100000）

- **容器崩溃处理**:
  - 自动重启机制：最多重试 3 次，指数退避（1秒、2秒、4秒）
  - 崩溃记录：日志记录到 `logs/container_crashes.log`
  - 用户提示：返回"工具执行环境需要重建，请重试"信息

- **输出截断**:
  - 如果命令输出超过 1MB，进行截断
  - 截断后在末尾添加 `...[truncated]` 标识

- **状态清理**:
  - 每次命令执行后清理临时文件（`/tmp/*`, `*.tmp`）
  - 每次命令执行后清理僵尸进程（后台运行的进程）
  - 保留环境变量和工作目录

- **工具名称**: `terminal`

### 2. Python 代码解释器 (Python REPL)

- **功能描述**: 赋予 Agent 逻辑计算、数据处理和脚本执行的能力。

- **沙箱方案**:
  - 使用 Docker 容器隔离，会话级持久容器（每个会话创建一个容器，会话期间复用）
  - 基础镜像：`python:3.11-slim`（带 Python 环境）

- **目录挂载**:
  - 用户指定的 `root_dir` 挂载到容器的 `/workspace`
  - 其他路径不可见，实现文件系统隔离

- **用户权限**:
  - 容器内使用非 root 固定用户（UID 1000）

- **网络隔离**:
  - 允许出站网络（支持 pip install、requests 等操作）

- **容器启动方式**:
  - 使用 Docker SDK（`python` 的 `docker` 包）

- **容器生命周期管理**:
  - 会话级持久容器：每个会话创建一个容器，会话期间复用
  - 容器创建时机：会话开始时（首次调用工具时）
  - 容器关闭时机：仅在会话结束时（用户关闭对话窗口、切换会话、删除会话等）
  - 不设置超时机制
  - 与 Agent 会话绑定：每个会话有独立的容器实例

- **容器关闭信号**:
  - 先发送 `SIGTERM` 信号优雅关闭（等待 10 秒）
  - 超时后发送 `SIGKILL` 信号强制终止

- **subprocess 禁止**:
  - 使用函数 hook 重写 `subprocess`、`os.system`、`os.popen` 等函数
  - 代码执行时尝试调用被禁止的函数会抛出 `SecurityError`
  - 错误信息明确告知被禁止的模块及替代方案

- **代码执行超时**:
  - 使用 `multiprocessing` 实现超时机制
  - 超时限制：30 秒
  - 超时后返回 `TimeoutError` 和原始 traceback

- **输出截断**:
  - 如果代码输出超过 1MB，进行截断
  - 截断后在末尾添加 `...[truncated]` 标识

- **错误处理**:
  - 返回完整的 Python traceback 信息
  - 让 Agent 自我分析和修正错误

- **状态清理**:
  - 每次代码执行后清理所有状态
  - 包括：临时变量、临时文件、导入的模块、工作目录
  - 每次执行都在全新的 Python 环境中运行

- **预装常用包**:
  - pandas：数据处理
  - numpy：数值计算
  - requests：HTTP 请求
  - matplotlib：数据可视化
  - beautifulsoup4：HTML 解析

- **资源限制**:
  - 内存：512MB
  - CPU：25%（cpu_quota=25000, cpu_period=100000）

- **容器崩溃处理**:
  - 自动重启机制：最多重试 3 次，指数退避（1秒、2秒、4秒）
  - 崩溃记录：日志记录到 `logs/container_crashes.log`
  - 用户提示：返回"工具执行环境需要重建，请重试"信息

- **工具名称**: `python_repl`

### 3. Fetch 网络信息获取

- **功能描述**: 用于获取指定 URL 的网页内容，Agent 联网的核心。

- **基础工具**: 直接使用 LangChain 内置工具：`langchain_community.tools.RequestsGetTool`（版本 0.4.1）

- **封装方式**: 创建 Wrapper 类，在基础工具上增强功能

- **HTML 清洗策略**:
  - 使用 BeautifulSoup 清洗 HTML 内容
  - 转换为 Markdown 格式
  - 仅抓取静态 HTML，不处理 JavaScript 渲染

  - **元素保留**: 保留基本结构（标题、段落、列表）和关键信息（链接、代码块、表格）
  - **元素去除**: 去除广告、导航、页脚、脚本、Cookie 提示等无关内容

  - **内容格式转换**:
    - **链接处理**: 转为 Markdown 链接格式 `[文本](URL)`
    - **代码块处理**: 转为 Markdown 代码块格式 ` ```python\ncode\n``` `
    - **表格处理**: 转为 Markdown 表格格式 `| 列1 | 列2 |\n| --- |\n| 值1 | 值2 | `

  - **无关内容去除**:
    - 导航栏: `nav, .navbar, .header-nav`
    - 侧边栏: `aside, .sidebar, .side-menu`
    - 页脚: `footer, .page-footer, .site-footer`
    - 广告横幅: `.ad-banner, .advertisement, .sponsor`
    - Cookie 提示: `.cookie-notice, .cookie-banner`
    - 分享按钮: `.share-buttons, .social-share`
    - 评论区域: `.comments, .comment-section`
    - 相关推荐: `.related-posts, .recommended`

- **黑白名单配置**:
  - 用户可通过 API 接口个性化配置域名白名单/黑名单、协议限制、URL 模式匹配
  - 默认为空（不限制）

- **请求配置**:
  - **请求超时**: 30 秒
  - **重试策略**: 失败后重试 3 次，每次返回错误信息
  - **User-Agent**: 固定使用标准浏览器 User-Agent（Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36）

- **响应大小限制**:
  - 按类型限制：允许文本内容，限制二进制文件（视频、图片等）
  - 检测 Content-Type，对非文本内容返回警告或拒绝

- **错误处理**:
  - 返回结构化错误响应
  - 包含错误类型、详细信息、修复建议
  - 错误类型包括：NetworkError、HTTPError、DomainBlockedError、TimeoutError、ContentTooLargeError
  - 让 Agent 能够理解错误原因并给出建议

- **来源引用**:
  - 工具返回结果自动包含 `source` 字段（来源 URL）
  - 不在 System Prompt 中硬编码

- **工具名称**: `fetch_url`

### 4. 文件读取工具 (File Reader)

- **功能描述**: 用于精准读取本地指定文件的内容。这是 Agent Skills 机制的核心依赖，用于读取本地文件。

- **技术实现**: 直接使用 LangChain 内置工具：`langchain_community.tools.file_management.ReadFileTool`（版本 0.4.1）

- **配置管理**:
  - **root_dir 配置**: 用户级别配置，不和 terminal 共享，初始默认值为项目根目录（backend/）
  - **配置更新**: 用户可通过 API 接口动态修改

- **文件类型过滤**: 仅允许文本文件（md, txt, py, js, json, yaml, yml 等）

- **安全检查**: 全部启用
  - **文件存在检查**: 检查文件是否存在
  - **权限检查**: 检查是否有读取权限
  - **路径安全**: 防止路径遍历攻击（`../`、符号链接逃逸）
  - **编码处理**: 使用 LangChain 工具内置的编码检测和处理

- **工具名称**: `read_file`

### 5. 文件写入工具 (File Writer)

- **功能描述**: 通用文件写入工具。

- **技术实现**: 直接使用 LangChain 内置工具：`langchain_community.tools.file_management.WriteFileTool`（版本 0.4.1）

- **配置管理**:
  - **root_dir 配置**: 与 read_file 共享同一配置
  - **配置更新**: 用户可通过 API 接口动态修改

- **文件类型过滤**: 仅允许文本文件（md, txt, py, js, json, yaml, yml 等）

- **安全检查**: 与 read_file 一致（文件存在、权限、路径安全、符号链接解析）

- **工具名称**: `write_file`

### 6. 检索记忆工具

- **功能描述**: 用于在长期记忆（已归档会话）中搜索与查询相关的内容，支持跨 session 的历史对话检索。

- **技术实现**:
  - 基于 LlamaIndex 的混合检索（BM25 + Vector）
  - 使用 RRF（Reciprocal Rank Fusion）融合两种检索结果
  - 索引存储于 `store/memory/` 目录
  - 向量存储使用 Chroma，docstore 持久化，cache 使用 SQLite

- **参数**:
  | 参数 | 类型 | 必需 | 说明 |
  |-----|------|------|------|
  | `query` | string | ✅ | 检索查询内容 |
  | `date_range` | tuple | ❌ | 日期范围过滤 (start_date, end_date)，格式 YYYY-MM-DD |

- **配置项**:
  - `top_k`: 从配置文件读取，默认 5

- **上下文集成**: 返回内容填充进当前的 user message，作为补充上下文

- **工具名称**: `search_memory`

### 7. 写入近端记忆工具

- **功能描述**: 用于将重要信息写入当天的近端记忆文件（`memory/YYYY-MM-DD.md`），记录临时性偏好、阶段性决策、对话摘要等近期上下文信息。

- **技术实现**:
  - 文件路径：`memory/YYYY-MM-DD.md`
  - 采用仅追加（append-only）策略
  - 自动处理日期、时间戳和格式
  - 确保写入原子性

- **参数**:
  | 参数 | 类型 | 必需 | 说明 |
  |-----|------|------|------|
  | `content` | string | ✅ | 要追加的内容（Markdown 格式） |
  | `category` | string | ❌ | 内容类别："对话摘要"、"重要事实"、"决策记录"，默认为"对话摘要" |
  | `date` | string | ❌ | 日期（YYYY-MM-DD 格式），默认为当天 |

- **文件格式示例**:
```markdown
# 2026-03-16 日志

## 对话摘要
- 用户提到正在开发 RAG 项目...

## 重要事实
- 用户正在开发***项目...

## 决策记录
- 2026-03-16：用户决定使用 SQLite...
```

- **注意**: Agent 在决定写入内容时，应避免与此前已写入的内容重复，防止冗余摘要

- **工具名称**: `write_near_memory`

### 8. 写入核心记忆工具

- **功能描述**: 用于将长期有效的重要信息写入核心记忆文件（位于 `core_memory/` 目录下），实现 Agent 的自我进化。

- **技术实现**:
  - 目标文件：`core_memory/{FILE_KEY}.md`
  - 禁止修改 AGENTS.md 和 SKILLS_SNAPSHOT.md（硬编码检查）
  - 写入时自动添加时间戳标记
  - 确保写入原子性

- **参数**:
  | 参数 | 类型 | 必需 | 说明 |
  |-----|------|------|------|
  | `file_key` | string | ✅ | 文件标识，可选值：`soul`、`identity`、`user`、`memory` |
  | `content` | string | ✅ | 要写入的内容（Markdown 格式） |
  | `mode` | string | ❌ | 写入模式：`append`（追加，默认）或 `replace`（替换） |

- **file_key 对应关系**:
  | file_key | 目标文件 | 用途 |
  |----------|----------|------|
  | `soul` | SOUL.md | 人格、语气、边界 |
  | `identity` | IDENTITY.md | 名称、风格、表情 |
  | `user` | USER.md | 用户画像、称呼方式 |
  | `memory` | MEMORY.md | 用户偏好、重要决策 |

- **安全限制**:
  - 禁止修改 `agents` 和 `skills_snapshot` 文件
  - 尝试修改时返回 `SecurityError`

- **工具名称**: `write_core_memory`
