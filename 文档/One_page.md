# SmartClaw - 轻量级透明记忆文件管理系统

## 一句话定位

让 AI Agent 的记忆像普通文件一样透明可控

---

## 核心设计哲学

### File-first Memory（文件即记忆）

摒弃不透明的向量数据库，回归最原始、最通用的 Markdown/JSON 文件系统。用户的每一次对话、Agent 的每一次反思，都以人类可读的文件形式存在。

### Skills as Plugins（技能即插件）

遵循 Anthropic 的 Agent Skills 范式，通过文件夹结构管理能力，实现"拖入即用"的技能扩展。

### 透明可控

所有 System Prompt 拼接逻辑、工具调用过程、记忆读写操作对开发者完全透明，拒绝"黑盒" Agent。

---

## 三层记忆架构

```
┌─────────────────────────────────────────────────┐
│  核心记忆 (Core Memory) - 6个 .md 文件          │
│  SOUL → IDENTITY → USER → MEMORY → AGENTS      │
│  定义 Agent 身份、人格、用户画像                 │
├─────────────────────────────────────────────────┤
│  近端记忆 (Near Memory) - 日记式 .md 文件       │
│  按日期存储，记录近期对话和决策                  │
│  自动加载最近 N 天内容                          │
├─────────────────────────────────────────────────┤
│  长期记忆 (Long-term Memory) - RAG 检索         │
│  会话归档后自动索引                             │
│  支持 BM25 + 向量混合检索                       │
└─────────────────────────────────────────────────┘
```

## 用户可见的文件结构

```
~/.smartclaw/
├── store/
│   ├── core_memory/     # 核心记忆（可直接编辑）
│   │   ├── SOUL.md
│   │   ├── IDENTITY.md
│   │   ├── USER.md
│   │   ├── MEMORY.md
│   │   ├── AGENTS.md
│   │   └── SKILLS_SNAPSHOT.md
│   ├── memory/          # 近端记忆（按日期）
│   │   └── 2024-01-15.md
│   └── rag/             # RAG 索引
├── sessions/            # 会话记录
├── skills/              # 用户自定义技能
└── config.yaml          # 配置文件
```

## 技术栈

| 组件 | 技术 | 说明 |
|-----|------|------|
| Agent 框架 | LangChain v1.0 + LangGraph | 工具调用决策、状态管理 |
| RAG 框架 | LlamaIndex | 文档索引、混合检索、向量存储 |
| Web 接口 | FastAPI + SSE | RESTful API、流式响应 |
| 向量存储 | Chroma + BM25 | 混合检索、RRF 融合 |
| 容器隔离 | Docker | 工具安全执行 |

## 与传统方案对比

| 特性 | SmartClaw | 传统 Agent 方案 |
|-----|-----------|----------------|
| 记忆可见性 | ✓✓✓ 纯文本文件 | ✗ 黑盒数据库 |
| 用户可编辑 | ✓ 直接改 .md | 有限 API 操作 |
| 部署复杂度 | 低 单机运行 | 高 多服务依赖 |
| 数据迁移 | 复制文件夹即可 | 需导出/转换 |
| 可审计性 | ✓ 版本控制友好 | ✗ 难以追溯 |
